# 后端 PRD 规则

> **权威性说明：** 本文件是 `doc/backend/backend-prd-rules.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

每个有意义的后端能力在实现前都必须有 PRD。

## 目的

后端 PRD 定义业务行为、数据、权限、状态流转、API 和运行约束。前端页面 spec 从后端 PRD 派生。

## 必需章节

每份后端 PRD 必须包含：

- 目标。
- 用户角色和身份要求。
- 用户动作。
- 权限规则。
- 数据模型影响。
- 状态机或状态值。
- API 合约。
- 错误场景。
- 审计和治理要求。
- 通知行为。
- 适用时的搜索索引行为。
- 测试期望。

## 与前端的关系

- 后端 PRD 不能规定具体视觉设计。
- 后端 PRD 可以定义前端必须暴露的数据和状态。
- 前端页面 spec 必须引用相关后端 PRD。
- 前端 spec 不能修改后端业务规则。

## 质量门槛

如果以下内容未定义，不接受 PRD：

- 谁可以执行该动作。
- 成功后发生什么。
- 失败后发生什么。
- 哪些数据是权威数据。
- 哪些状态对用户可见。
- 哪些动作需要审计日志。
