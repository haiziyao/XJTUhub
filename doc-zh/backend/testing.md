# 后端测试

> **权威性说明：** 本文件是 `doc/backend/testing.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

后端测试必须关注业务边界和高风险集成。

## 必需测试层级

- 领域规则单元测试。
- 应用服务模块测试。
- 外部可见行为 API 测试。
- 可行时对存储适配器和 Elasticsearch 索引做集成测试。

## 必需覆盖区域

高风险区域需要测试：

- 权限检查。
- 匿名可追溯。
- 资源审核标识和下载策略。
- 会员展示状态。
- 板块发布策略。
- 举报和点踩行为。
- 对象存储适配器行为。
- 搜索索引和重建索引。
- 敏感动作的审计日志创建。

## 测试数据

测试应包含：

- 邮箱用户。
- 校园认证用户。
- 大会员。
- 版主。
- 管理员。
- 匿名内容。
- 未审核资源。
- 已审核资源。
- 有争议资源。

## 回归规则

修复 bug 时，除非该 bug 纯粹是配置或文档问题，否则应添加一个没有修复时会失败的测试。
