# 前端工作流

> **权威性说明：** 本文件是 `doc/frontend/frontend-workflow.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

前端工作遵循 PRD 到视觉参考的流程。

## 必需流程

1. 编写或引用后端 PRD。
2. 从 PRD 派生前端页面需求。
3. 从页面需求生成前端视觉提示词。
4. 使用 gpt-image-2 为重要页面状态生成参考图。
5. 筛选参考图，并将其回灌到实现提示词中。
6. 基于页面 spec、视觉参考和 API 合约生成或实现前端代码。

## 硬性规则

- 后端 PRD 存在前，不要自由设计核心业务页面。
- 不要让视觉图片替代需求。
- 不要只根据图片实现。
- 每个已实现页面都必须追溯到页面 spec 和 API 合约。
- 如果某页面不需要视觉参考，必须在页面 spec 中说明原因。

## 后端 PRD 提供什么

后端 PRD 提供：

- 用户动作。
- 权限。
- 状态机。
- 数据字段。
- API 行为。
- 错误状态。
- 审计和治理要求。

后端 PRD 不提供：

- 最终视觉风格。
- 组件布局。
- 字体选择。
- 图片提示词。

## 前端 spec 提供什么

前端 spec 提供：

- 页面目标。
- 用户流程。
- API 依赖。
- UI 状态。
- 桌面端和移动端行为。
- 可访问性要求。
- 视觉提示词要求。

## 图片生成

当视觉质量重要时，使用 gpt-image-2 生成重要前端参考图。

生成足够状态以覆盖：

- 默认内容。
- 空状态。
- 加载状态。
- 错误状态。
- 无权限状态。
- 审核或警告状态。
- 移动端布局。
- 相关时的弹窗或抽屉。
