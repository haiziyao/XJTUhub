# 图片提示词规则

> **权威性说明：** 本文件是 `doc/frontend/image-prompt-rules.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

图片提示词从页面 spec 生成，而不是从模糊想法生成。

## 提示词输入

每个提示词应使用：

- 页面 spec。
- 目标视口。
- 必需状态。
- 真实内容示例。
- UI 规则。
- 品牌和语气约束。

## 提示词输出

提示词应要求为以下状态生成参考图：

- 桌面端默认状态。
- 移动端默认状态。
- 空状态或引导状态。
- 错误或无权限状态。
- 相关时的审核、警告或可信标识状态。
- 相关时的弹窗、抽屉或编辑器状态。

## 可追溯性

每张生成图片都应可追溯到：

- 后端 PRD。
- 页面 spec。
- 提示词文本。
- 目标状态。

## 在实现中的使用

- 图片是视觉参考，不是事实源。
- 如果图片与 spec 冲突，以 spec 为准。
- 如果图片揭示了更好的交互，先更新页面 spec，再实现。
