# 前端实现规则

> **权威性说明：** 本文件是 `doc/frontend/implementation-rules.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

前端使用 Nuxt 3。

## API 访问

- 使用共享 API client 层。
- 不要从随意的组件直接调用后端服务。
- 不要直接调用 Elasticsearch、MinIO、MySQL、Redis 或邮件服务。
- 一致处理后端错误码。

## SSR 和 CSR

- 当索引、分享或首屏阅读性能重要时，公开内容页使用 SSR。
- 对 SSR 价值较低的高交互认证页面使用 CSR。
- 在页面 spec 中记录例外。

## 权限

- 后端 API 必须提供权限和 capability 信息。
- 前端可以隐藏不可用控件，但后端仍必须强制执行权限。
- 不要只基于角色名硬编码特权行为。

## 组件

为以下内容创建可复用组件：

- 内容卡片。
- 资源审核标识。
- 会员标志。
- 匿名作者标识。
- 互动控件。
- 举报控件。
- 空、加载、错误和无权限状态。

## 视觉参考

实现提示词必须包含：

- 页面 spec。
- API 合约。
- 已选视觉参考图，或不需要图片的文档化原因。
- 相关 UI 规则。
