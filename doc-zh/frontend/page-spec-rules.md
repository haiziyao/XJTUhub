# 页面 Spec 规则

> **权威性说明：** 本文件是 `doc/frontend/page-spec-rules.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

每个非平凡前端页面在实现前都必须有页面 spec。

## 必需章节

每份页面 spec 必须包含：

- 页面目标。
- 来源后端 PRD。
- 用户角色和认证状态。
- API 依赖。
- 主布局。
- 用户动作。
- 加载状态。
- 空状态。
- 错误状态。
- 无权限状态。
- 桌面端行为。
- 移动端行为。
- 可访问性说明。
- 视觉参考要求。

## 与后端的关系

- 页面 spec 可以引用后端 PRD 和 API 合约。
- 页面 spec 不能修改业务规则。
- 如果前端需求暴露出后端缺口，先更新或创建后端 PRD。

## 状态覆盖

内容页需要覆盖：

- 正常已发布内容。
- 匿名内容。
- 大会员作者。
- 未审核资源。
- 已审核资源。
- 已删除或隐藏内容。
- 无权限。

管理页需要覆盖：

- 列表状态。
- 筛选状态。
- 详情状态。
- 确认状态。
- 失败状态。
- 审计可见性。
