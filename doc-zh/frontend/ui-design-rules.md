# UI 设计规则

> **权威性说明：** 本文件是 `doc/frontend/ui-design-rules.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

XJTUhub 前端使用 Nuxt 3。

## 渲染策略

- 公开内容页应优先使用 SSR。
- 登录后的强交互页面可以使用 CSR。
- 管理页可以使用 CSR，除非需要公开索引。
- 不要在没有文档化理由的情况下把所有页面都做成纯 CSR，尤其当公开索引和分享预览仍然重要时。

## SEO 和分享

公开内容页应支持：

- 有意义的标题。
- 描述。
- 适用时的 canonical URL。
- 分享预览元数据。
- 尽可能可读的服务端渲染内容。

## 必需视觉状态

UI 必须清晰表达：

- 匿名内容。
- 大会员红名和标志。
- 未审核资源的警告或红色标识。
- 已审核资源的可信或绿色标识。
- 被拒绝或有争议的资源状态。
- 相关时的用户角色和权限限制。
- 加载、空、错误和无权限状态。

## 颜色和可访问性

- 红色和绿色标识必须包含文字或图标辅助。
- 不要只依赖颜色。
- 交互控件必须有清晰的焦点和禁用状态。
- 移动端布局不能隐藏治理或资源可信标识。

## 组件规则

- 共享 API 状态展示应组件化。
- 权限敏感控件应尽可能根据后端提供的 capability flags 渲染。
- 不要只在前端硬编码角色检查。
- 除非页面 spec 另有要求，搜索、板块列表和个人列表使用统一内容卡片。
