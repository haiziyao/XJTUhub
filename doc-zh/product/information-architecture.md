# 信息架构

> **权威性说明：** 本文件是 `doc/product/information-architecture.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

本文档定义初始内容结构和导航规则。

## 顶层板块

Phase 1 从少量宽泛板块开始：

- 讨论。
- 资源。
- 经验。
- 活动与竞赛。
- 工具与项目。
- 站务。

Phase 1 不要创建过多顶层板块。在需求被验证前，优先使用标签和子板块。

## 标签

标签是早期扩展的首选机制。

示例：

- 课程。
- 学院。
- 保研。
- 就业。
- 竞赛。
- 社团。
- 软件。
- 工具。
- 新生。

标签可以跨内容类型共享。

## 统一内容类型

平台使用带类型的统一内容模型：

- `post`：普通讨论。
- `resource`：资源共享。
- `activity`：活动和通知。
- `experience`：前辈经验。
- `blog`：校园博客或知识分享。
- `tool`：工具推荐或项目介绍。

## 导航规则

- 顶层导航必须保持稳定且数量少。
- 新增顶层板块需要产品理由和文档化决策。
- 搜索结果应尽可能使用统一内容卡片，并显示类型特定标识。
- 资源可信标识必须出现在列表页和详情页。
- 匿名和会员状态在相关位置必须可见。
