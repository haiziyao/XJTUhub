# 阶段路线图

> **权威性说明：** 本文件是 `doc/product/phase-roadmap.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

本路线图描述实现阶段。它不是对所有后续功能一定会建设的承诺。

## Phase 1：社区内容基础

目标：构建一个可用的社区平台，具备内容、资源、搜索、基础治理和通知能力。

范围：

- Nuxt 3 前端。
- Spring Boot 模块化单体后端。
- MySQL 作为事实数据源。
- Redis 支撑缓存、会话和限流。
- 通过存储抽象使用 MinIO。
- 从一开始使用 Elasticsearch 做内容搜索。
- 邮箱注册，以及规划中的校园官方 APP 扫码登录。
- 使用统一内容模型承载帖子、资源、活动、经验、博客和工具。
- 中等治理。
- 站内通知和重要邮件通知。
- 基础管理后台，覆盖用户、内容、举报、资源、板块、标签、公告和审计日志。

## Phase 2：模块深化

目标：在核心闭环可用后，深化高价值内容模块。

可能范围：

- 更强的资源库，包含课程、教师、学期、版本和评分字段。
- 更好的活动和竞赛聚合。
- 更细化的经验分享模板。
- 更完善的通知订阅和邮件偏好。
- 更丰富的治理工作流。
- 更好的 Elasticsearch 相关性、同义词和中文搜索调优。
- 更成熟的前端视觉流程，为每个页面状态生成参考图。

## Phase 3：生态扩展

目标：扩展到核心社区平台之外。

可能范围：

- 校园工具集成。
- 社团和组织管理。
- 推荐和语义搜索。
- 更高级的数据分析。
- 移动端或桌面客户端。
- 在硬件压力或性能需要时，将选定后端模块拆分到 Rust Axum。

## 延后规则

- 不要实现 Phase 2 或 Phase 3 功能，除非它们直接支撑 Phase 1 稳定性。
- 保留扩展点，但不要过早建设完整系统。
- 未来模块必须先有自己的 PRD 再实现。
