# 产品约束

> **权威性说明：** 本文件是 `doc/product/product-constraints.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

本文档定义 XJTUhub 的产品边界。后端和前端文档可以引用这些规则，但不能静默改写它们。

## 产品定位

XJTUhub 是面向校园场景的讨论、资源共享和信息聚合平台。

Phase 1 必须聚焦一个核心闭环：

1. 用户发现板块和内容。
2. 用户发布讨论内容或进行互动。
3. 用户在相关场景中附加或查找资源。
4. 用户关注板块、接收通知并回访。

## 优先级顺序

产品优先级为：

1. 社区讨论。
2. 资源共享。
3. 信息聚合。
4. 校园工具、社团、竞赛和高级服务。

不要让低优先级模块拖慢 Phase 1 的社区/内容基础建设。

## 身份

- 允许邮箱注册，不要求 XJTU 邮箱域名。
- 校园官方 APP 扫码登录是首选高可信身份来源。
- 身份可信等级必须由后端建模，不能由前端推断。
- 未来校园局域网部署可以降低暴露面，但不能替代认证、权限、治理、审计日志和限流。

## 会员

会员与角色、身份可信等级相互独立。

Phase 1 会员行为：

- 大会员昵称显示为红色。
- 大会员显示可见的大会员标志。
- 会员不授予治理、管理、置顶、删除、审核或排序权限。
- 隐藏、封禁、删除或被撤销权益的账号不能展示有效会员权益。

## 匿名内容

- 匿名发布只表示前端匿名。
- 后端必须保留真实作者，用于审计、举报、治理和防滥用。
- 匿名内容必须支持点踩交互。
- 点踩是治理和质量信号，不是直接删除投票。

## 治理等级

XJTUhub 使用中等治理：

- 普通帖子和评论可以发布后可见。
- 资源、置顶内容、公告、敏感板块和官方组织内容需要明确的审核状态。
- 举报和治理动作必须可追溯。

## 资源可信标识

上传资源可以立即下载。

- 未审核资源必须显示红色或警示标识。
- 已审核资源必须显示绿色或可信标识。
- 标识必须包含文字或图标辅助，不能只依赖颜色。
- 被拒绝或有争议的资源由后端策略控制。

## Phase 1 非目标

- 完整校园工具套件。
- 完整社团管理系统。
- 复杂数据分析报表。
- 个性化推荐引擎。
- Flutter 或 Tauri 多客户端应用。
