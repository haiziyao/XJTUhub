# 决策日志

> **权威性说明：** 本文件是 `doc/shared/decision-log.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

在这里记录长期产品和工程决策。

## 2026-04-23：Phase 1 优先级

决策：Phase 1 优先社区讨论，其次资源共享，再次信息聚合。

原因：这能形成最短的可用闭环，同时为资源、活动、工具、社团和经验分享保留空间。

## 2026-04-23：前端框架

决策：使用 Nuxt 3。

原因：XJTUhub 可能沉淀公开内容、经验文章、资源索引和可分享页面。Nuxt 3 支持基于 Vue 的 SSR 和 CSR 混合。

## 2026-04-23：后端架构

决策：Phase 1 使用 Spring Boot 模块化单体。

原因：模块化单体保持部署简单，同时保留未来拆分边界。

## 2026-04-23：未来后端拆分

决策：Rust Axum 是未来压力点的可选方案。

原因：部分模块未来可能需要更高性能或更低资源占用，但过早拆服务会拖慢 Phase 1。

## 2026-04-23：搜索

决策：Phase 1 使用 Elasticsearch。

原因：搜索对内容、资源和信息聚合至关重要。MySQL 仍是事实数据源。

## 2026-04-23：对象存储

决策：使用对象存储抽象，MinIO 作为 Phase 1 适配器。

原因：项目已有 MinIO，但未来迁移阿里 OSS 或腾讯 COS 不应影响业务模块。

## 2026-04-23：前端视觉工作流

决策：前端实现遵循后端 PRD -> 页面 spec -> 图片提示词 -> gpt-image-2 参考图 -> 代码实现。

原因：这让前端设计基于后端行为，同时允许在编码前进行高质量视觉探索。
