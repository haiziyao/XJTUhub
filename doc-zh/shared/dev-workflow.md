# 开发流程

> **权威性说明：** 本文件是 `doc/shared/dev-workflow.md` 的中文镜像，只供人工阅读。AI agent 和实现工作必须以英文 `doc/` 为准。

XJTUhub 使用文档先行开发。

## 通用流程

1. 需要时定义或更新产品约束。
2. 为能力编写后端 PRD。
3. 编写或更新 API 和数据合约。
4. 派生前端页面 spec。
5. 需要时生成视觉提示词和参考图。
6. 按已批准文档实现后端和前端。
7. 添加测试。
8. 更新 `TODO.md` 和决策文档。

## 前后端隔离

- 后端文档位于 `doc/backend`。
- 前端文档位于 `doc/frontend`。
- 共享术语和流程文档位于 `doc/shared`。
- 允许交叉引用。
- 混淆业务规则和视觉实现的文档应拆分。

## 修改规则

修改约束时：

1. 更新归属文档。
2. 如果变更影响架构、工作流、安全或产品范围，添加决策日志。
3. 更新受影响文档。
4. 在实现说明或 PR 描述中提及该变更。

## 分支和提交建议

尽可能使用聚焦的提交。

建议提交前缀：

- `docs:`
- `feat:`
- `fix:`
- `test:`
- `refactor:`
- `chore:`

## 评审指导

评审者应检查：

- 变更是否遵循相关 PRD/spec。
- 前后端职责是否隔离。
- 权限是否在后端强制执行。
- 供应商 SDK 是否隔离在适配器后。
- 高风险行为是否包含测试。
