# Information Architecture

This document defines initial content structure and navigation rules.

## Top-Level Boards

Phase 1 starts with a small number of broad boards:

- Discussion.
- Resources.
- Experience.
- Activities and competitions.
- Tools and projects.
- Site affairs.

Do not create many top-level boards in Phase 1. Prefer tags and sub-boards when the need is proven.

## Tags

Tags are the preferred early expansion mechanism.

Examples:

- Course.
- College.
- Graduate recommendation.
- Employment.
- Competition.
- Club.
- Software.
- Tool.
- Freshman.

Tags may be shared across content types.

## Unified Content Types

The platform uses a unified content model with typed content:

- `post`: ordinary discussion.
- `resource`: resource sharing.
- `activity`: activities and notices.
- `experience`: senior experience.
- `blog`: campus blog or knowledge sharing.
- `tool`: tool recommendation or project introduction.

## Navigation Rules

- Top-level navigation must stay stable and small.
- A new top-level board requires a product reason and a documented decision.
- Search results should use a unified content card shape with type-specific labels.
- Resource trust labels must appear in list and detail views.
- Anonymous and membership states must be visible where relevant.
