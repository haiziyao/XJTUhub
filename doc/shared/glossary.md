# Glossary

## Content

A unified published object. It may represent a post, resource, activity, experience, blog, or tool.

## Board

A navigational and policy container for content.

## Tag

A flexible classification label shared across boards and content types.

## Auth Level

The trust level of a user's identity, such as email user or campus APP verified user.

## Role

The authorization role that controls moderation and administration capability.

## Membership

A display and future benefits layer. Phase 1 premium membership gives a red name and premium badge.

## Anonymous Content

Content whose author is hidden in ordinary frontend views while remaining traceable in backend audit and moderation workflows.

## Review Status

The trust or moderation state of content or resource attachments.

## Resource

A content type for shared files, links, course materials, or other downloadable/reference material.

## PRD

Product requirements document. In this project, backend PRDs define capability, data, permissions, API behavior, and state transitions.
